Phase 0: Stabilize Core (1-2 days)
Goal: make Admin side reliable before adding new pages.

Fix route ownership and wiring.
Ensure Admin routes are mounted and protected with role checks.
Remove/replace broken Shared booking service dependencies.
Normalize slot status values and booking lifecycle transitions.
Validate Admin parking scoping using authenticated user context.
Exit criteria:

Admin can fetch only their parking bookings.
Admin entry/exit APIs return correct responses.
No unauthorized access to booking APIs.

Add validation matrix:
wrong code
wrong parking
cancelled/expired

already exited

Frontend MVP pages:

Ticket Verification
Entry Desk
Exit Desk
